# Contributing to BountySleuth

Thank you for your interest in contributing to **BountySleuth**! We welcome bug reports, feature requests, and pull requests from everyone. 

## How to Contribute

### 1. Reporting Bugs
Before creating bug reports, please check the existing issues as you might find out that you don't need to create one. When you are creating a bug report, please include as many details as possible:
* Use a clear and descriptive title
* Describe the exact steps to reproduce the problem
* Provide specific examples or URLs where the extension fails
* Include your browser version (Chrome/Firefox) and BountySleuth version

### 2. Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an enhancement request:
* Use a clear and descriptive title
* Provide a step-by-step description of the suggested enhancement
* Explain why this enhancement would be useful to most bug bounty hunters

### 3. Pull Requests
* **Fork the repository** and create your branch from `main`.
* If you've added new features, please document them in the `README.md`.
* Ensure that the code follows the existing style and conventions.
* Test your changes on both Chrome and Firefox if possible.
* Issue that pull request!

## Development Setup
Check the `README.md` for standard installation instructions on loading the unpacked extension in Developer Mode. Both `chrome_bounty_extension` and `firefox_bounty_extension` share identical or near-identical codebases. 

Thank you for helping make BountySleuth better!
